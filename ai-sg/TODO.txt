 Лучше наверное вместо spot-точек искать направление движения(нормализованный вектор), 
 что в принципе аналогично spot-точкам, но более логично и нет дополнительных вопросос по дистанции spot-точек.
 В отличие от поиска spot-точек как матрицы точек, поиск направлений производится по лучам сектора с определённым шагом угла(допустим 5 градусов).
 Поиск направлений должен осуществляться с каким-то временным интервалом, зависящим от скиллов игрока.
 
 Действия игрока и состояния - суть одно и то же. Переход игрока из состояния в состояние аналогично смене действий.
 Как осуществлять переход между состояниями?:
	Решение о смене состояния принимается в самом состоянии, при этом учитывается текущее состояние игрока.
	Для выбора состояния сначала проводится анализ вариантов новых состояний на предмет получения рейтинга будущего состояния,
	потом на основе рейтинга происходит переход в новое состояние или сохрвнение старого (если его рейтинг выше).
	Допустим игрок находится в состоянии - "принять мяч" (игрок с мячом). 
	Варианты для анализа cостояния:
		- проброс мяча себе на ход для резкого ускорения (бросок вперёд с мячом)
		- проброс мяча себе на ход для спокойного движения вперёд
		- стоять с мячом (блокировать мяч)
		- пас
		- навес
		- удар
	При этом каждое из анализируемых состояний(или действий) может иметь разные варианты (напр адресаты пасов),
	поэтому каждое состояние анализируется сначала на поиск лучшего варианта состояния, а потом только сравниваются
	между собой.
	
	Анализ действий в CI:
	
		// Если мяч в поле у игрока - действие игрока с мячом
        if ($Match->Ball->ball_field === 'field')
        {                                                     
            // Оценка действий игрока с мячом
            $arr_rating = array();                  // массив оценок действий игрока 

            // Оценка поражения ворот 
            array_push($arr_rating, rateKick());
            // если мяч на земле (в ногах игрока)
            if($Match->Ball->z === 0)
            {           
                // если игрок не вратарь
                if ($Match->Ball->ball_gamer !== 0)
                {
                    // Оценка дриблинга
                    array_push($arr_rating, rateDribbling());
                    // Оценка движения вперёд
                    array_push($arr_rating, rateMove());
                }      
                // Оценка позииций игроков на пасы в зону ногой
                array_push($arr_rating, ratePassZone());
                // Оценка позииций игроков на пасы назад
                array_push($arr_rating, ratePassBack());
                // Оценка позииций игроков на навесы
                array_push($arr_rating, rateHeadPass());
                // Оценка позииций игроков на навесы в зону
                array_push($arr_rating, rateHeadPassZone());                                                                                  
            }
            // Оценка позииций игроков на пасы в ноги
            array_push($arr_rating, ratePass());

            // Оценка позииций игроков на пасы в недодачу
            array_push($arr_rating, ratePassUnder());

            // Выбор действия с наибольшей оценкой    
            $arr_action = findMaxRating($arr_rating); 

            // Вынос мяча 
            if (empty($arr_action['rating']) or $arr_action['rating'] < RATING_PASS_OUT)
               $arr_action = ratePassOut();

            // Расчёт действия игрока с мячом
            getActionGamerBall($arr_action);  

	В getActionGamerBall($arr_action) фактически производятся установки действий, что близко к заданию состояний.
	Но в CI не учитывается текущее состояние игрока, вернее оно размазано по коду.
